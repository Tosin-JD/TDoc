# Contributing to TDoc

First off, thank you for considering contributing to TDoc! It's people like you that make TDoc such a great tool for the Android community.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs

- **Check if the bug has already been reported** by searching on GitHub under [Issues](https://github.com/oluwatosin/TDoc/issues).
- If you can't find an open issue addressing the problem, [open a new one](https://github.com/oluwatosin/TDoc/issues/new).
- Include a **clear title and description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- **Check if the enhancement has already been suggested**.
- [Open a new issue](https://github.com/oluwatosin/TDoc/issues/new) which clearly describes the enhancement.

### Pull Requests

1. **Fork the repository**.
2. **Create a new branch** (`git checkout -b feature/your-feature-name`).
3. **Make your changes**.
4. **Add tests** for your changes.
5. **Ensure the test suite passes**.
6. **Commit your changes** (`git commit -am 'Add some feature'`).
7. **Push to the branch** (`git push origin feature/your-feature-name`).
8. **Create a new Pull Request**.

## Style Guide

- Follow the [Kotlin Style Guide](https://kotlinlang.org/docs/coding-conventions.html).
- Use meaningful variable and function names.
- Write clear, concise comments where necessary.
- Ensure all public APIs are documented.
